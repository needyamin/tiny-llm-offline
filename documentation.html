Tiny LLM API — documentation

Offline-ready GPT-Neo 125M text generation via FastAPI, default port 22122. Runtime does not call Hugging Face Hub when HF_HUB_OFFLINE=1.

Hardware Overview Layout Prerequisites Setup Environment API Docker Troubleshooting

Hardware requirements (read first)

ResourceMinimumRecommendedNotes
CPUx86_64, 2+ cores4+ coresDefault image is CPU inference; GPU not required.
RAM8 GB (tight)16 GBOS + Docker + PyTorch + model peak.
Storage~10 GB free15 GB+ SSDImage + deps + model + margin.
GPU—OptionalNot used by default Dockerfile.

Below minimum: risk of out-of-memory, failed installs, or very slow generation. Training/fine-tuning needs more RAM and disk than this service alone.

What it does / does not

  • Does: load weights from a local folder; generate text; list/read files in data/.
  • Does not: train or fine-tune on your dataset (add a separate training pipeline if needed).

Setup step by step

1) Download model (one machine with internet)

pip install huggingface_hub
python scripts/download_model.py

Alternative: huggingface-cli download EleutherAI/gpt-neo-125M --local-dir models/gpt-neo-125m. Copy models/gpt-neo-125m to the offline server.

2) Dataset (optional)

Place files under data/. Allowed extensions: .txt, .md, .jsonl, .csv.

3) Docker (production-style)

docker compose up --build -d

Build step needs network for pip unless you import a pre-built image. Runtime uses ./models → /models, ./data → /data (read-only).

4) Local Python (no Docker)

pip install torch --index-url https://download.pytorch.org/whl/cpu
pip install -r requirements.txt
set HF_HUB_OFFLINE=1
set TRANSFORMERS_OFFLINE=1
uvicorn app.main:app --host 0.0.0.0 --port 22122

Linux/macOS: use export instead of set. Defaults: model models/gpt-neo-125m, data data/ next to app/.

API reference

Base URL: http://<host>:22122  ·  JSON for POST bodies (Content-Type: application/json).

Interactive docs from the running server: /docs (Swagger UI), /redoc (ReDoc), OpenAPI schema at /openapi.json.

GET /health

Liveness and configuration snapshot. No request body.

HTTPMeaning
200JSON body below.

Response fields (JSON object):

FieldTypeDescription
statusstringAlways "ok" when the handler runs.
model_pathstringDirectory passed to Transformers (host path or container /models/…).
offlinebooleantrue if HF_HUB_OFFLINE is 1 / true / yes.
cudabooleantrue if PyTorch sees a CUDA device (default image is CPU-only).
cpu_threadsintegerTORCH_NUM_THREADS (default 4).
interop_threadsintegerTORCH_INTEROP_THREADS (default 1).
data_dirstringResolved dataset directory (DATA_DIR).

Example

curl -s http://localhost:22122/health
{
  "status": "ok",
  "model_path": "/models/gpt-neo-125m",
  "offline": true,
  "cuda": false,
  "cpu_threads": 4,
  "interop_threads": 1,
  "data_dir": "/data"
}

POST /v1/generate

Causal LM text generation. Returns the pipeline’s generated_text (prompt + continuation).

Request body (JSON object):

FieldTypeDefaultConstraintsDescription
promptstring—required; length 1…MAX_PROMPT_CHARS (default 6000)Input text.
max_new_tokensinteger961…256New tokens to generate (not total context length).
temperaturenumber0.80.1…2.0Sampling temperature.
top_pnumber0.950.0…1.0Nucleus sampling.
do_samplebooleantrue—If false, decoding is greedy (ignores temperature/top_p).

Response (200):

FieldTypeDescription
textstringFull generated string (includes the original prompt).
modelstringSame as model_path on the server.

Errors

HTTPWhen
422Validation error (bad types, prompt empty/too long, out-of-range numbers). Body: FastAPI/Pydantic error detail.
503"Model not loaded" (should not occur after successful startup).

Examples

curl -s http://localhost:22122/v1/generate ^
  -H "Content-Type: application/json" ^
  -d "{\"prompt\":\"Once upon a time\",\"max_new_tokens\":64,\"temperature\":0.8}"

Linux/macOS: one line with -d '{"prompt":"Hello"}'.

{
  "text": "Hello ...",
  "model": "/models/gpt-neo-125m"
}

GET /v1/dataset/files

Lists non-recursive files in DATA_DIR whose extension is allowed.

Allowed file extensions: .txt, .md, .jsonl, .csv (case-insensitive).

Response (200):

FieldTypeDescription
data_dirstringDataset directory path.
filesarrayObjects {"name": "<filename>", "size": <bytes>}, sorted by name.
errorstringPresent only if DATA_DIR is missing: "DATA_DIR missing".
curl -s http://localhost:22122/v1/dataset/files

GET /v1/dataset/file

Reads one text file from DATA_DIR by basename only (no subfolders).

Query parameter

NameRequiredDescription
nameyesFilename only. Pattern: starts with alphanumeric; then letters, digits, _, ., - only (no / or ..).

Response (200): name (string), size (integer, bytes), text (string, UTF-8 with replacement on invalid bytes).

Errors

HTTPDetail
400"Invalid file name" or "Invalid path"
404"File not found" (wrong extension, missing file, not a file).
413"File too large" — file size > MAX_DATA_READ_BYTES (default 2 MiB).
curl -s "http://localhost:22122/v1/dataset/file?name=sample.txt"

Troubleshooting

  • Startup error “MODEL_PATH is not a directory” — create models/gpt-neo-125m with a full HF snapshot.
  • Incomplete model — need config.json, tokenizer files, and weight file (.safetensors or .bin).
  • Slow replies — normal on CPU; lower max_new_tokens.
